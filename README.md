# Venusescort — Frontend

**A standards-based collective of untamed muses.**

This is the public-facing web experience for Venusescort: a high-class collective where curation, discretion, and a clear philosophy sit at the centre. The frontend is built to feel like the House itself—confident, minimal, and unmistakably intentional.

---

## The Experience

- **One threshold.** Visitors are met with an age gate so that only those who belong can enter. No clutter, no noise—just a single, clear choice before the door opens.

- **Hero & disciplines.** The landing experience leads with the brand and three ways into the House: **Kink**, **Somatics**, and **Femme**. Each discipline has its own definition, tone, and space—reflected in typography, layout, and copy.

- **Muses.** The heart of the site. Muses are presented as cards with imagery and overlays; on the home page they appear in a filtered carousel by discipline, and on the dedicated Muses page as a responsive grid. Each muse has a full profile: intro, structured profile, erotic preferences, and a “My Perfect Date” section—all designed to read like a dossier, not a listing.

- **Manifesto.** A dedicated page for the House’s principles. Numbered, serif-led, and built for slow reading. The content is driven by the API so the manifesto can evolve without touching the frontend.

- **Contact.** A two-column layout: information and accordions on one side (booking, payment, agency hours), contact channels and a message form on the other. Clear, calm, and conversion-minded.

---

## What It’s Made Of

- **Next.js** (App Router) for routing, server components, and metadata.
- **Tailwind CSS** for a consistent, design-token–driven look (noir, cream, serif, tracking).
- **Swiper** for the home-page muse carousel and the muse-detail image lightbox.
- **Open Graph & Twitter Cards** so shared links (especially muse profiles) show the right title, description, and image.

The UI is built to feel editorial and restrained: serif headlines, generous spacing, dark overlays on imagery, and copy that speaks in the House’s voice. No setup instructions here—this README is an introduction. The rest is in the code and the API.
